# Git Review Release Rules

## 1. Commit Rules
1. 커밋 메시지 형식은 `type(scope): imperative summary`를 사용한다.
2. 커밋은 목적 단위로 작게 유지한다.
3. 강하게 결합된 변경(씬/스크립트/리소스)은 한 커밋으로 묶는다.
4. 코드/콘텐츠 수정 후 같은 작업 사이클에서 즉시 커밋/푸시한다.

## 2. Branch and Push Rules
1. 기본 브랜치는 `main`을 사용한다.
2. 로컬 완료 상태로 장시간 방치하지 않는다.
3. 푸시 전 스모크 결과와 핵심 변경 파일을 다시 확인한다.

## 3. Review Rules
1. 리뷰 우선순위는 버그, 리스크, 회귀, 테스트 누락 순이다.
2. 리뷰 기록은 `docs/reviews/`에 남긴다.
3. UI/비주얼 변경은 캡처/GIF 근거를 포함한다.

## 4. PR Rules
1. 변경 요약과 영향 경로를 명시한다.
2. 수동 테스트 단계와 결과를 포함한다.
3. 이슈/태스크 링크를 연결한다.

## 5. Release Candidate Rules
1. QA 게이트 통과 없이 릴리즈 후보 태깅 금지.
2. 주요 리스크가 남아 있으면 릴리즈 노트에 명시한다.
