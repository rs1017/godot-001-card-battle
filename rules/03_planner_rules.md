# Planner Rules

## 1. Document Language and Frame
1. 기획 문서 기본 언어는 한국어다.
2. 문서 시작부에 프레임 타입을 명시한다.
3. 기본 템플릿은 `docs/plans/templates/game_plan_format_kr.md`를 따른다.

## 2. Planning Scope Rules
1. 한 문서는 단일 기능 스코프를 우선한다.
2. 대형 주제는 기능 단위 문서로 분해한다.
3. 각 기능 문서는 실행 가능한 수준의 규칙/변수/예외를 포함해야 한다.

## 3. Mandatory Sections
1. 표지와 목차를 포함한다.
2. 문제 관찰/근거/가설을 해결안보다 먼저 작성한다.
3. `fixed`(고정) vs `variable`(가변) 항목을 분리한다.
4. 변경 의사결정 로그를 포함한다.
5. UI 문서는 화면 목록, 사용자 흐름, 컴포넌트/상태/이벤트, 예외 처리, QA 수용 기준을 포함한다.

## 4. Data and Assets Rules
1. 본문은 `.md`, 구조 데이터는 `.csv`로 분리한다.
2. CSV는 `docs/plans/data/`에 저장한다.
3. 기획 이미지는 `docs/plans/images/`에 저장한다.
4. 외부 이미지 URL 직접 삽입을 금지한다.

## 5. Scenario and System Rules
1. 시나리오 중심 문서는 4막(기/승/전/결) 요약을 포함한다.
2. 전투/스킬 문서는 프로그래머 인수인계 스펙(흐름, 규칙, 변수, 예외, 기술 요구사항)을 포함한다.
3. 스킬 문서는 애니메이션/VFX/SFX 타이밍 가이드를 포함한다.
4. 맵 문서는 개요, 연결 그래프, 범례, 맵 파라미터, 지역 배경 설정, 랜드마크 표를 포함한다.

## 6. Validation Rules
1. 기획 승인 없이 개발 착수를 금지한다.
2. 승인 반려 시 참조 재수집 -> 재기획 순서로 되돌린다.
